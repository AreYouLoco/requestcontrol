Request Control Rules

Define rules for controlling HTTP requests. Requests that match an active rule, will be intercepted and the selected action for the rule is applied.

Show Help

Pattern

Request pattern consists of URL scheme, host and path.

Scheme

Scheme matches the request URL protocol. It can either match http or https alone or both protocols at the same time.

Host

Host is the domain name part in the request URL. Host in pattern can match

1. A complete hostname. www.example.com
2. The given host and any of its subdomains. example.com and check the include subdomains checkbox to match any subdomain (e.g. www.example.com, good.example.com)
3. The given host and all listed top-level domains. (can be combined with 2.) www.example.* and write the top-level domains to the top-level domain name list (e.g. com, co.uk).
4. Any host. *

Path

Path is optional
